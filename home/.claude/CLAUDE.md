# Global Claude Instructions

## 環境（マシン構成）

- **開発機は2台**：この **Mac mini M1** と **Mac mini M4**。dotfiles（`nakahironobu/dotfiles`）と Claude 横断設定はこの2台で共有する。
- **iMac は開発に使わない**：子供の閲覧用。SSD 容量が大きいので Google Drive 共有フォルダの「置き場」として使っているだけ。
  - Google Drive の「その他のパソコン/マイ iMac/…」は、その iMac のバックアップ鏡＝**単なるストレージ**であって開発機ではない（だから他機から書込み・移動・削除しないルールが効く。元データの集約はコピーで、削除は iMac 本人で）。

## 大きなタスクを受け取ったとき

複数のステップにわたる作業や、コンテキストが長くなりそうなタスクを受け取った場合、サスペンド（コンテキスト圧縮・セッション切断）が発生する前に `/save-knowledge` の実施をユーザーに勧める。

**判断基準:** 以下のいずれかに当てはまる場合は作業の区切りで提案する。
- 複数ファイルの編集・生成を伴う
- 検品・レビュー作業など人間の操作を挟むループがある
- 1セッションで完結しない可能性が高い（point72〜76 の連続処理など）

**提案タイミング:** タスク完了時、または大きな区切り（1ポイント処理完了など）ごとに一言添える。

## Python は終始一貫 uv を使う

Python の実行・依存管理・仮想環境は **常に `uv`** を使う。`pip` / `python -m venv` / `python` 直叩き / `conda` を素で使わない（環境がぶれる原因）。

- **単発実行:** `uv run python ...` / `uv run <script>.py`（プロジェクト直下なら依存を自動解決）。
- **仮想環境:** 作成は `uv venv`。依存追加はプロジェクトなら `uv add`、一時的なら `uv pip install`。ロックは `uv sync`。
- **既存プロジェクト:** `pyproject.toml` / `uv.lock` があればそれに従う。新規も最初から uv 前提で始める（`uv init`）。
- **外部ツール（例: ndlocr-lite）も専用の uv venv を作って動かす**。システム Python や素の pip にインストールしない。
- Python バージョンも uv に管理させる（`uv python install` / `--python` 指定）。

## Claude.app 内で長時間プロセスを起動しない

Claude Code の Bash ツール（および Claude.app 配下の zsh）から `uvicorn`/`fastapi`/`http.server` 等の長時間サーバーを起動すると、そのプロセスは **Claude.app のサンドボックス権限を継承する**。具体的に踏みやすいのは:

- 外部ボリューム（`/Volumes/LaCie` など）の `iterdir()` / `rglob()` が `PermissionError [Errno 1] Operation not permitted` で落ちる、または silent に空を返す
- `~/Documents` `~/Desktop` 等の保護フォルダへのアクセスが拒否される
- マイク・画面収録などのデバイス系 API が一切叩けない

**判断基準:**
- 「永続化されたサーバー」「録画/録音/ファイルスキャン」を伴うプロセスは、Claude の Bash で起動しない。**ユーザーに Terminal.app から起動するよう依頼する**
- 短時間の単発スクリプト（`uv run python -c ...` で数秒で終わるもの）は Claude Bash で OK

**診断手順:** 既存サーバーが怪しい挙動をするとき、`ps -o pid,ppid,command -p <pid>` で親プロセスを遡り、最終的に `Claude.app` に行き着くなら権限不足を最優先で疑う。`lsof -ti:<port> | xargs kill -9` で停止 → Terminal.app から再起動して切り分ける。

## リポジトリ と クラウドストレージ の線引き

コード・設定・**ユーザーに見せる成果物**は Git（GitHub）で一元管理し、**元ファイル（著作物・大容量の原本）と再生成可能な中間生成物**は Google Drive に置いて共有する。

- **リポジトリは Google Drive の外（ローカル）に clone する。** `.git` が Drive 同期と競合して破損する／成果物バイナリが Drive のオンデマンド削除で 0 バイト化する、のを避けるため。
- **判断基準:** ユーザー向けに表示・配布される成果物 → Git。元データ・中間生成物（スクリプトで再生成できるもの）→ Drive。
- **パス参照は分離する:** リポジトリから Drive 側を参照する際は環境変数 or シンボリックリンクを使い、未設定ならローカルにフォールバック（カタカナを含む「マイドライブ」パスはシンボリックリンクで英語別名にすると扱いやすい）。
- **危険な置き場に注意:** Google Drive の「その他のパソコン/＜PC名＞/…」は特定PCのバックアップ鏡。他機から書込み・移動・削除しない（同期事故の原因）。集約はコピーで行い、元の削除はそのPC上で行う。FileMaker（.fmp12）等の同期に弱いファイルは Drive に置かずローカルに残す。
